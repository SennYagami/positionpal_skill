# Skill: position_safety_monitor

Analyze crypto account for position risk. Returns a structured safety report covering account summary, macro market conditions, per-position risk scores, and actionable recommendations.

---

## Base URL

```
BASE_URL = https://loyal-celebration-production-e4d8.up.railway.app
```

All endpoints below are relative to this base.

---

## Authentication

The system supports two authentication methods:

### Method 1: API Key + Secret (Direct - for Agents)

Use exchange API credentials directly in the request body:

```bash
curl -s -X POST "${BASE_URL}/api/analyze/agent" \
  -H "Content-Type: application/json" \
  -d '{"api_key":"<EXCHANGE_KEY>","api_secret":"<EXCHANGE_SECRET>","exchange":"binance"}'
```

### Method 2: JWT Token + API Keys (for Humans + Agents)

**For Humans:** Connect wallet (MetaMask) → Sign message → Get JWT → Manage API Keys

**For Agents:** Use API Key generated by human user:

```bash
curl -s -X POST "${BASE_URL}/api/analyze/agent" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: psm_xxxxxxxxxxxx" \
  -d '{"exchange":"binance"}'
```

## Endpoints

### 0. POST /api/analyze/agent — Agent Optimized (Recommended for Agents)

**Designed for AI agents** — returns a concise, pre-structured response with top 10 positions only. Use this instead of `/api/analyze` for better parsing and presentation.

**Request body**

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `exchange` | string | yes | — | `binance`, `hyperliquid`, or `aster` |
| `api_key` | string | no* | — | Exchange API key (for binance/aster) |
| `api_secret` | string | no* | — | Exchange API secret (for binance/aster) |
| `address` | string | no* | — | Wallet address (for hyperliquid) |
| `account_id` | string | no | — | Account ID (when using JWT/API Key auth) |
| `analysis_depth` | `"basic"` \| `"detailed"` | no | `"basic"` | `basic` = rule-based; `detailed` = OpenAI-enhanced |

* Either `api_key+api_secret` or `address` or `account_id` is required.

**Supported Exchanges**

| Exchange | Auth Type | Fields Required |
|---|---|---|
| `binance` | API Key + Secret | `api_key`, `api_secret` |
| `hyperliquid` | Wallet Address | `address` |
| `aster` | API Key + Secret | `api_key`, `api_secret` |

**Response: `AgentAnalyzeResponse`**

```json
{
  "report_id": "2026-02-26T03:27:04.502Z",
  "overall_score": 65,
  "level": "risky",
  "total_assets_usd": 12345.67,
  "position_count": 15,
  "top_positions": [
    {
      "symbol": "SOL",
      "side": "spot",
      "value_usd": 4500.00,
      "allocation_percent": 36.5,
      "risk_score": 78,
      "risk_level": "high",
      "main_risk": "OI激增35%+超买"
    }
  ],
  "recommendations": [
    "建议减仓 SOL (风险分 78)",
    "建议观望 ETH",
    "建议持有 BNB, USDT"
  ]
}
```

**Response fields:**

| Field | Type | Description |
|---|---|---|
| `overall_score` | number | 0–100 portfolio risk score |
| `level` | string | `safe` / `cautious` / `risky` / `dangerous` |
| `total_assets_usd` | number | Total portfolio value |
| `position_count` | number | Total number of positions |
| `top_positions` | array | Top 10 positions sorted by risk_score (desc) |
| `top_positions[].symbol` | string | Asset symbol |
| `top_positions[].side` | string | `spot` / `long` / `short` |
| `top_positions[].value_usd` | number | Position value in USD |
| `top_positions[].allocation_percent` | number | Portfolio allocation % |
| `top_positions[].risk_score` | number | 0–100 risk score |
| `top_positions[].risk_level` | string | `low` / `medium` / `high` / `critical` |
| `top_positions[].main_risk` | string | One-line risk summary |
| `recommendations` | array | Top 3 action items |

**curl example**

```bash
curl -s -X POST "${BASE_URL}/api/analyze/agent" \
  -H "Content-Type: application/json" \
  -d '{"exchange":"binance","api_key":"<KEY>","api_secret":"<SECRET>","analysis_depth":"basic"}'
```

### 10. POST /api/analyze/stream — Streaming Analysis (SSE)

Same analysis as `/api/analyze` but streams progress events via **Server-Sent Events**.

**Request body** — identical to `/api/analyze`

**Response**: `text/event-stream`. Each event is a JSON line prefixed with `data: `.

| Event type | Fields | Notes |
|---|---|---|
| Progress | `stage`, `progress` (0–100), `message` | Emitted throughout analysis |
| Done | `stage: "done"`, `progress: 100`, `report_id`, `report` | Final event |
| Error | `stage: "error"`, `error` | Emitted on failure |

**curl example**

```bash
curl -s -N -X POST "${BASE_URL}/api/analyze/stream" \
  -H "Content-Type: application/json" \
  -d '{"exchange":"binance","api_key":"<KEY>","api_secret":"<SECRET>"}'
```

## Agent Workflow Patterns

### Pattern 1: Direct Exchange API (for Agents)

```python
import requests

BASE_URL = "https://loyal-celebration-production-e4d8.up.railway.app"

# Analyze directly with exchange credentials
result = requests.post(f"{BASE_URL}/api/analyze/agent", json={
    "exchange": "hyperliquid",
    "address": "0xd47587702a91731Dc1089B5DB0932cF820151A91",
    "analysis_depth": "basic"
}).json()

print(f"Risk: {result['level']} ({result['overall_score']})")
for pos in result['top_positions'][:3]:
    print(f"  {pos['symbol']}: {pos['risk_score']} ({pos['risk_level']})")
```

### Pattern 2: Using System API Key (for Agents)

```python
# Analyze using stored account with API Key
result = requests.post(f"{BASE_URL}/api/analyze/agent", headers={
    "X-API-Key": "psm_xxxxxxxxxxxx"
}, json={
    "account_id": "uuid-of-stored-account"
}).json()
```

### Pattern 3: User Account Management (for Humans)

```python
# Login with wallet (get JWT)
login_result = requests.post(f"{BASE_URL}/api/auth/wallet", json={
    "walletAddress": "0x...",
    "signature": "0x...",
    "message": "..."
}).json()

jwt = login_result["token"]

# Create API Key for agent
api_key = requests.post(f"{BASE_URL}/api/auth/keys", headers={
    "Authorization": f"Bearer {jwt}"
}, json={
    "label": "My Trading Bot",
    "permissions": "read"
}).json()

print(f"API Key: {api_key['key']}")
print(f"Secret: {api_key['secret']}")  # Save this!

# Add exchange account
account = requests.post(f"{BASE_URL}/api/accounts", headers={
    "Authorization": f"Bearer {jwt}"
}, json={
    "exchange": "hyperliquid",
    "label": "Main Account",
    "address": "0xd47587702a91731Dc1089B5DB0932cF820151A91"
}).json()
```

## Security

- Exchange API keys are encrypted before storage.
- JWT tokens expire after 7 days.
- API Keys can be revoked at any time.
- Only read-only exchange permissions required.
- Wallet signature verification ensures only wallet owner can access account.
